# Week 01 — Day 01 (Agenda)

## Goals
- Everyone can access GitHub
- Everyone has a fork of the course repository
- Everyone has a local clone
- Everyone can open `index.html`
- Everyone sees JavaScript output in the browser Console
- Everyone edits `script.js` at least once
- (Optional) First commit and push

---

## Opening
- Course welcome
- Day 1 focus: setup and orientation
- Overview of using a single GitHub workflow throughout the semester

---

## Typical class structure
- Tuesdays: demos and guided work
- Thursdays: in-class work and commit/push
- Most JavaScript work is completed during class

---

## Developer Tools sanity check
- Open browser Developer Tools
- Open the Console
- Enter a simple JavaScript expression
- Confirm output appears

---

## GitHub setup
- Open the instructor’s repository
- Fork the repository to create a personal copy
- Clone the fork locally
- Open the repository in a code editor
- Verify the `week-01` folder is present

---

## Run the project
- Open `week-01/index.html` in the browser
- Open the Console
- Confirm JavaScript output appears

---

## Demo: updating from the instructor repository
- Introducing the concept of remotes
- Instructor makes a change in the main repository
- Demonstration of pulling updates from the instructor repository into your own fork

---

## First edit
- Open `week-01/script.js`
- Modify the console message
- Save and reload the page
- Confirm updated output in the Console

---

## (Optional) Commit and push
- Check repository status
- Stage changes
- Commit changes
- Push to personal fork

## Close
